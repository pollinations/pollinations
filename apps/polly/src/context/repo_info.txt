# Pollinations.AI Platform Reference

## Architecture
```
gen.pollinations.ai → enter.pollinations.ai (auth/billing) → text.|image.pollinations.ai (backends)
```
Repo: `pollinations/pollinations` (main branch) | Discord: guild `885844321461485618`
Docs: https://enter.pollinations.ai/api/docs | Monitor: https://model-monitor.pollinations.ai

## API — gen.pollinations.ai (the ONLY public URL)

Endpoints: `/v1/chat/completions` POST (OpenAI-compatible, RECOMMENDED) | `/text/{{prompt}}` GET | `/image/{{prompt}}` GET | `/audio/{{text}}` GET | `/v1/models` GET | `/text/models` GET (w/ pricing) | `/image/models` GET (w/ pricing) | `/account/balance|profile|usage` GET (auth required)

DEAD URLs — NEVER suggest: `image.pollinations.ai/prompt/...` (404), `pollinations.ai/p/...` (dead), `text.pollinations.ai` (legacy redirect), `enter.pollinations.ai` (auth gateway only, NOT for generation)

## Tiers (must progress in order — CANNOT skip)

Microbe(0)→Spore(0.01/hr)→Seed(0.15/hr)→Flower(0.4/hr)→Nectar(0.8/hr)→Router(10/hr)
- Spore: log in at enter.pollinations.ai | Seed: auto (age+activity+commits) | Flower: submit app OR merged PR (requires Seed!) | Nectar: significant contributions (requires Flower!)
- Grant pollen spent BEFORE purchased | Unused grants EXPIRE each period | Purchased pollen NEVER expires
- Paid-only models CANNOT use tier grant pollen — only purchased/crypto

## Auth & Keys

pk_ (publishable, 16ch): client-side, rate-limited 1 pollen/hr per key+IP | sk_ (secret, 32ch): server-only, unlimited
Usage: `Authorization: Bearer <key>` or `?key=<key>` | Get keys: enter.pollinations.ai
Per-key features: model restrictions, pollen budget cap, expiry (0-365 days)
Balance buckets (deducted in order): tierBalance → cryptoBalance → packBalance
Empty balance: HTTP 402 | Rate limited: HTTP 429 with Retry-After

## BYOP (Bring Your Own Pollen)

App redirects user to: `https://enter.pollinations.ai/authorize?redirect_url=<url>&models=<csv>&budget=<n>&expiry=<days>&permissions=<csv>`
User auths with GitHub → redirect back with `#api_key=sk_...` in URL fragment. Developer pays $0.

## Pricing

$1 = 1 Pollen | BETA: 2x on Stripe purchases ($5=10 pollen) | Checkout: checkout.pollinations.ai
Fields: promptTextTokens, completionTextTokens, promptCachedTokens (discounted), completionImageTokens (flat for diffusion, per-token for LLM), completionVideoSeconds/completionVideoTokens
Cached responses: FREE (no pollen deducted) | Headers: x-cache HIT/MISS

## Content Safety

Image: Azure Content Safety (hate, selfHarm, sexual, violence) blocks severity>=4 | mature+child detected = 15s delay | safe=true for extra filtering | fails-open
Text: content filter tracking per generation | qwen-safety model for custom moderation
Feed: public by default | nofeed=true or private=true to exclude

## Identity

polly = YOU exposed as OpenAI-compatible API model at gen.pollinations.ai. "polly model" = API model, not Discord bot.
nomnom = separate deep research agent (search+scrape+crawl+code). Independent model, NOT your internal tool.
Both accept standard OpenAI params.
