# Website Foundation + Legal Walking Skeleton Implementation Plan (spec Phase 2 + legal slice of Phase 3)

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Cut `pollinations.ai/` over to the new stack â€” React 19 + Tailwind v4 (`@tailwindcss/vite`) + TanStack Router (file-based) + `PolliProvider` â€” wired to `@pollinations/ui`/`@pollinations/sdk` via `file:` deps and prebuild hooks, and render the **legal pages on the `Prose` primitive** as the end-to-end validation. The other routes render temporary placeholders until their own plans (3â€“8) rebuild them.

**Architecture:** Mirror the proven setup in `enter.pollinations.ai/frontend` (TanStack Router file-based routes, `@tailwindcss/vite`, single `@pollinations/ui/app.css` entry) and `apps/model-monitor` (full-width `polli-ui-shell` body, `predev`/`prebuild` package builds). The Cloudflare Worker (`src/worker.ts`) and `wrangler.toml` are **unchanged** â€” the worker keeps owning crawler SEO meta via HTMLRewriter; TanStack `head:`/`HeadContent` provides client-side tab titles. `PolliProvider` (from `@pollinations/sdk/react`) wraps the router and owns auth; no app in this repo wired it before, so the provider config here is the new reference.

**Tech Stack:** React 19.2, Tailwind v4.1 (`@tailwindcss/vite`), `@tanstack/react-router` + `@tanstack/router-plugin` ^1.139, `@pollinations/ui@0.0.3` (Plan 1) + `@pollinations/sdk@5` via `file:` deps, Vite 7, Cloudflare Workers (wrangler, unchanged).

**Prerequisite:** Plan 1 (`@pollinations/ui@0.0.3` with `Prose`/`Textarea`/`FileUpload`) is merged/built. This plan consumes `Prose`.

**Conventions / hard facts (verified):**
- Current scripts: `dev`, `build` (`tsc && vite build`), `build:staging`, `build:production`, `deploy:{staging,production}` (deploy calls `build:*`, **not** `build`). No prebuild hooks exist today.
- `npm` pre-hooks are **name-exact** â†’ we add `prebuild:staging` + `prebuild:production` (+ `predev`/`prebuild`), each calling `build:ui`. This matches `enter`, not `model-monitor` (whose bare `prebuild` works only because it has no `build:*`).
- Body shell class is **`polli-ui-shell`** (full-width, `height:100dvh; overflow:hidden`) â€” so the document does not scroll; an inner `<main>` owns scroll. Per-route `data-theme` goes on the route container, never `<body>`.
- **Tailwind classes: author UNPREFIXED utilities in app code** (`flex h-full`, `min-h-full`, `text-theme-text-strong`, `font-heading`). The `polli:` prefix is **package-internal only** â€” `app.css` ships an unprefixed `@import "tailwindcss"` + unprefixed `@theme` tokens, and Tailwind generates utilities by scanning the consumer's source. âš ď¸Ź The route/component code blocks below were originally written with a `polli:` prefix; that is **wrong** and silently no-ops (it broke the inner-scroll container in review). They were corrected to unprefixed during execution. Reference: `apps/model-monitor` (zero `polli:` in app code).
- `pollinations.ai/` is a **standalone npm project** (its own `package-lock.json` + `node_modules`), **not** a workspace member â€” run `npm install`/`npm run *` **inside** `pollinations.ai/`, with package deps as `file:../packages/ui` / `file:../packages/sdk` (like `apps/model-monitor`). (Earlier draft wrongly said `-w pollinations.ai` from root.)
- Run app commands from `pollinations.ai/`. Cloudflare worker + `wrangler.toml` are out of scope (do not edit).

## Task 5: Root route (scroll container + head)

The root owns the full-viewport layout. Because `polli-ui-shell` makes `<body>` non-scrolling, the root provides the scrolling `<main>`. `HeadContent` renders per-route `<title>`/meta. (The app shell header/footer arrives in Plan 3; here `<main>` is the whole canvas.)

**Files:** Create `src/routes/__root.tsx`.

- [ ] **Step 1: Create `src/routes/__root.tsx`**

```tsx
import { createRootRoute, HeadContent, Outlet } from "@tanstack/react-router";

export const Route = createRootRoute({
    head: () => ({
        meta: [{ title: "pollinations.ai" }],
    }),
    component: RootLayout,
});

function RootLayout() {
    return (
        <div className="polli:flex polli:h-full polli:flex-col">
            <HeadContent />
            {/* Shell body is overflow-hidden; this <main> is the scroll owner. */}
            <main
                id="app-scroll"
                className="polli:flex-1 polli:overflow-y-auto"
            >
                <Outlet />
            </main>
        </div>
    );
}
```

> `id="app-scroll"` is the named inner scroll container. Plan 3's header scroll-hide hook listens to this element (not `window`), per the shell consequence recorded in the spec.
