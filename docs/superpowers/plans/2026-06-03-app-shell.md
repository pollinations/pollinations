# App Shell Implementation Plan (spec Phase 3 — Header / Footer / UserMenu)

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Give `pollinations.ai/` a real app shell — a sticky scroll-aware `Header` (brand + nav + account menu), a `Footer` (branding + legal + social + Enter), and a `UserMenu` composed from `@pollinations/ui/auth/sdk` + `@pollinations/ui/wallet/sdk` — replacing the `<main>`-only root layout from Plan 2. Auth is the SDK redirect/api-key model via the already-mounted `PolliProvider`.

**Architecture:** The package ships **no** shell/Header/Footer/Logo/Avatar components — compose them locally in `src/components/shell/`. Auth/balance come from the SDK-bound module subpaths (`@pollinations/ui/auth/sdk`: `LoginButton`/`LogoutButton`/`UserAvatar`/`UserName`/`WhenLoggedIn`/`WhenLoggedOut`; `@pollinations/ui/wallet/sdk`: `Balance`) — they read `PolliProvider` context, no props/wiring. Nav uses TanStack `<Link>` with `activeProps`. The shell is a fixed-theme (`data-theme="green"`) chrome; route content overrides theme per-route inside `#app-scroll`. The `Header` is `position: sticky` **inside `#app-scroll`** and hides on scroll-down / shows on scroll-up via a hook that reads the **`#app-scroll` container** (never `window` — the shell body is `overflow:hidden`).

**Tech Stack:** React 19, `@tanstack/react-router` 1.170.11, `@pollinations/ui@0.0.3` (primitives + `/auth/sdk` + `/wallet/sdk`), `@pollinations/sdk@5` react hooks, Tailwind v4.

**CSS convention (hard rule — see spec):** app code uses **UNPREFIXED** Tailwind utilities (`flex h-full`, `text-theme-text-strong`, `border-theme-border`). The `polli:` prefix is package-internal only; prefixed classes in app code silently no-op. Reference: `apps/model-monitor`.

**Verified facts (from research):**
- `@pollinations/ui/auth/sdk` components return `null` when their condition fails (e.g. `LoginButton` → null when logged in; `UserAvatar` → null when logged out or no image+name). They require a `<PolliProvider>` ancestor (already in `main.tsx`).
- `LogoutButton` / `LoginButton` props = `Omit<ButtonProps<"button">, "as"|"onClick"|"children"> & { children }` — they own `onClick` (do **not** pass `onClick`).
- `Dropdown` props: `theme` is **required**; `trigger: (open) => ReactNode`; `children` may be `(close) => ReactNode`; `align?: "start"|"end"`.
- `Button` is polymorphic (`as`), `theme?`, `size?: "small"|"medium"|"large"`, only `intent?: "danger"`. `IconButton` has **no** `theme` and **requires** `onClick`.
- `Balance` (`@pollinations/ui/wallet/sdk`) renders a `Chip` `"<n> pollen"`, self-fetching; needs no props and no `wallet.css`.
- Icons (`GitHubIcon`/`DiscordIcon`/`XIcon`) accept `className` (SVG props).
- Brand asset: `@pollinations/ui/assets/logo-wordmark.svg`; default import is a URL string (`vite-env.d.ts` references `vite/client`). Render via CSS `mask` + `bg-current` so it inherits `text-theme-text-strong` (the enter pattern).
- `__root.tsx` (Plan 2) currently: `<div data-theme? no> flex h-full flex-col > HeadContent + <main id="app-scroll" flex-1 overflow-y-auto> Outlet`. Nav routes already exist: `/ /play /apps /community` (+ `/terms /privacy /refunds /docs`).

## Task 2: `useHideOnScroll` hook

**Files:** Create `src/components/shell/useHideOnScroll.ts`.

- [ ] **Step 1: Write the hook**

```ts
import { useEffect, useState } from "react";

/**
 * Returns `true` when a sticky header should hide. Reads scroll from the
 * element `#${scrollElementId}` (NOT window — the shell body is
 * overflow:hidden and `#app-scroll` owns scroll). Hides on scroll-down,
 * shows on scroll-up and always near the top.
 */
export function useHideOnScroll(scrollElementId: string, threshold = 16): boolean {
    const [hidden, setHidden] = useState(false);

    useEffect(() => {
        const el = document.getElementById(scrollElementId);
        if (!el) return;
        let lastY = el.scrollTop;
        const onScroll = () => {
            const y = el.scrollTop;
            if (y < threshold) {
                setHidden(false);
            } else if (Math.abs(y - lastY) > 4) {
                setHidden(y > lastY);
            }
            lastY = y;
        };
        el.addEventListener("scroll", onScroll, { passive: true });
        return () => el.removeEventListener("scroll", onScroll);
    }, [scrollElementId, threshold]);

    return hidden;
}
```

**Files:** Create `src/components/shell/UserMenu.tsx`.

- [ ] **Step 1: Write the component**

```tsx
import { Button, Dropdown } from "@pollinations/ui";
import {
    LoginButton,
    LogoutButton,
    UserAvatar,
    UserName,
    WhenLoggedIn,
    WhenLoggedOut,
} from "@pollinations/ui/auth/sdk";
import { Balance } from "@pollinations/ui/wallet/sdk";
import { ENTER_HREF } from "./links.ts";

export function UserMenu() {
    return (
        <>
            <WhenLoggedOut>
                <LoginButton theme="green" size="small">
                    Login
                </LoginButton>
            </WhenLoggedOut>

            <WhenLoggedIn>
                <Dropdown
                    theme="green"
                    align="end"
                    trigger={() => (
                        <button
                            type="button"
                            aria-label="Account menu"
                            className="flex items-center gap-2 rounded-full border border-theme-border bg-theme-bg-subtle px-2 py-1 text-theme-text-strong"
                        >
                            <UserAvatar size="sm" />
                            <UserName className="hidden text-sm font-semibold sm:block" />
                        </button>
                    )}
                >
                    {(close) => (
                        <div className="flex w-56 flex-col gap-3 p-3">
                            <div className="flex items-center gap-3">
                                <UserAvatar size="md" />
                                <UserName className="text-sm font-semibold text-theme-text-strong" />
                            </div>
                            <div className="flex items-center justify-between text-sm text-theme-text-base">
                                <span>Balance</span>
                                <Balance />
                            </div>
                            <Button
                                as="a"
                                href={ENTER_HREF}
                                target="_blank"
                                rel="noopener noreferrer"
                                theme="green"
                                size="small"
                                onClick={close}
                            >
                                Dashboard
                            </Button>
                            <LogoutButton theme="green" size="small">
                                Log out
                            </LogoutButton>
                        </div>
                    )}
                </Dropdown>
            </WhenLoggedIn>
        </>
    );
}
```

> `LogoutButton` owns its `onClick` (calls `logout()`), so the dropdown isn't closed by it — fine, logout clears state immediately. `Dashboard` is a plain `Button as="a"` so it accepts `onClick={close}`.

## Task 4: `Header`

**Files:** Create `src/components/shell/Header.tsx`.

- [ ] **Step 1: Write the component**

```tsx
import { cn } from "@pollinations/ui";
import wordmarkUrl from "@pollinations/ui/assets/logo-wordmark.svg";
import { Link } from "@tanstack/react-router";
import { NAV_ITEMS } from "./links.ts";
import { UserMenu } from "./UserMenu.tsx";
import { useHideOnScroll } from "./useHideOnScroll.ts";

export function Header() {
    const hidden = useHideOnScroll("app-scroll");
    return (
        <header
            className={cn(
                "sticky top-0 z-30 border-b border-theme-border bg-theme-bg-pale/90 backdrop-blur transition-transform duration-300",
                hidden && "-translate-y-full",
            )}
        >
            <div className="mx-auto flex h-14 max-w-5xl items-center justify-between gap-4 px-4 sm:px-6">
                <Link to="/" aria-label="pollinations.ai home" className="flex items-center">
                    <span
                        className="block h-5 w-44 bg-current text-theme-text-strong"
                        style={{
                            maskImage: `url(${wordmarkUrl})`,
                            WebkitMaskImage: `url(${wordmarkUrl})`,
                            maskRepeat: "no-repeat",
                            WebkitMaskRepeat: "no-repeat",
                            maskSize: "contain",
                            WebkitMaskSize: "contain",
                            maskPosition: "left center",
                            WebkitMaskPosition: "left center",
                        }}
                    />
                </Link>

                <nav className="flex items-center gap-1">
                    {NAV_ITEMS.map((item) => (
                        <Link
                            key={item.to}
                            to={item.to}
                            activeOptions={{ exact: item.exact }}
                            className="rounded-md px-2 py-1.5 font-body text-sm font-semibold text-theme-text-soft hover:text-theme-text-strong sm:px-3"
                            activeProps={{ className: "text-theme-text-strong" }}
                        >
                            {item.label}
                        </Link>
                    ))}
                </nav>

                <UserMenu />
            </div>
        </header>
    );
}
```

> `bg-theme-bg-pale/90` + `backdrop-blur` gives a translucent sticky bar. The wordmark is a CSS mask filled with `currentColor` (= `text-theme-text-strong`) so it adapts to the shell theme and can't be invisible. Mobile: nav is compact (`gap-1`, smaller padding); a hamburger/drawer is deferred — at the `max-w-5xl` width the 4 short labels fit narrow viewports.

## Task 6: Wire the shell into `__root.tsx`

`Header` is sticky **inside** `#app-scroll` so the hide-on-scroll transform works without leaving a layout gap. `Footer` sits at the bottom of the scrolled content; a `min-h-full` flex column keeps it at the viewport bottom on short pages.

**Files:** Modify `src/routes/__root.tsx`.

- [ ] **Step 1: Replace the file**

```tsx
import { createRootRoute, HeadContent, Outlet } from "@tanstack/react-router";
import { Footer } from "../components/shell/Footer.tsx";
import { Header } from "../components/shell/Header.tsx";

export const Route = createRootRoute({
    head: () => ({
        meta: [{ title: "pollinations.ai" }],
    }),
    component: RootLayout,
});

function RootLayout() {
    return (
        <div data-theme="green" className="flex h-full flex-col">
            <HeadContent />
            {/* Shell body is overflow-hidden; this <main> is the scroll owner. */}
            <main id="app-scroll" className="flex-1 overflow-y-auto">
                <Header />
                <div className="flex min-h-full flex-col">
                    <div className="flex-1">
                        <Outlet />
                    </div>
                    <Footer />
                </div>
            </main>
        </div>
    );
}
```

> `data-theme="green"` makes the shell chrome brand-green; route containers inside `<Outlet/>` set their own `data-theme` (play=violet, etc.) and override for content. `#app-scroll` stays the scroll owner — the Plan 2 inner-scroll contract is preserved (verified in Task 7).
